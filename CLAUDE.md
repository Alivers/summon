# Summon — CLAUDE.md

macOS menu bar app that lets you launch terminal apps (claude, lazygit, k9s, …) via global hotkeys without needing an open terminal first.

## Build & run

```bash
# Generate Xcode project (required after project.yml changes)
xcodegen generate

# Build + install to ~/Applications (stable path = Accessibility permission survives restarts)
xcodebuild -scheme Summon -destination "platform=macOS" \
  -derivedDataPath /tmp/summon-build build && \
  cp -R /tmp/summon-build/Build/Products/Debug/Summon.app ~/Applications/

# Run (kill previous instance first)
pkill -x Summon 2>/dev/null; sleep 0.5; open ~/Applications/Summon.app
```

> **Note on Accessibility permission**: macOS ties the permission to the binary hash.
> Rebuilding replaces the binary → permission must be re-granted once after each build.
> Between restarts of the *same* binary the permission persists.
> Permanent fix: sign with a Developer ID certificate (future milestone).

No Swift Package Manager — dependencies (SwiftTerm, HotKey) are declared in `project.yml` and resolved by Xcode/xcodebuild.

## Architecture

| Layer | File(s) | Role |
|---|---|---|
| Entry point | `App/SummonApp.swift` | `@main`, wires AppDelegate via `@NSApplicationDelegateAdaptor` |
| App lifecycle | `App/AppDelegate.swift` | Menu bar setup, rebuilds menu on slot changes via Combine |
| Data model | `Models/SlotConfig.swift` | `SlotConfig` (command + workingDir + hotKey + windowSize), `HotKeyConfig` with Carbon modifier flags |
| Session state | `Managers/SessionManager.swift` | `@MainActor ObservableObject`; owns all `TerminalSession`s; persists slots to JSON in `~/Library/Application Support/Summon/slots.json` |
| Global hotkeys | `Managers/HotKeyManager.swift` | Carbon Event API (`RegisterEventHotKey`); uses file-scope globals to bridge C callback into Swift |
| Session lifecycle | `Terminal/TerminalSession.swift` | show/hide toggle per slot; re-launches when process exits |
| Terminal window | `Terminal/TerminalWindowController.swift` | `NSWindowController` + SwiftTerm `LocalProcessTerminalView`; launches via login shell (`-l`) so PATH is correct |
| Settings UI | `Views/SettingsView.swift` | SwiftUI `NavigationSplitView`; add/edit/remove slots |

## Key conventions

- **Carbon modifier flags** (not `NSEvent.ModifierFlags`): `cmdKey=256, shiftKey=512, optionKey=2048, controlKey=4096`. Stored in `HotKeyConfig.modifierFlags`.
- **Process launch**: always `shell -l -c "cd 'dir' && exec command"` — login shell ensures user PATH (brew, rbenv, nvm, etc.) is loaded.
- **Window level** is `.normal` (not `.floating`) — brought to front on summon via `makeKeyAndOrderFront`, behaves as a regular window otherwise.
- **`isReleasedWhenClosed = false`** on all terminal windows — closing hides, doesn't destroy the session.
- **`LSUIElement = true`** in Info.plist — no Dock icon, menu bar only.
- **Sandbox disabled** in entitlements — required for global hotkeys and spawning arbitrary processes.

## Project file management

The `.xcodeproj` is **gitignored** and generated by xcodegen. After any change to `project.yml`, run `xcodegen generate`. Never edit the `.xcodeproj` directly.
